Getting Sources

If you want to get the sources of Xsemantics, for instance, to contribute new features and patches, you can get them via Git.

We use Buckminster as the building and materialization infrastructure; this will let you automatically materialize the sources in the workspace and setup the target platform, so that you will have everything that is necessary to have a working workspace with Xsemantics sources.

Install Buckminster in the IDE

First of all, you need to install Buckminster in your Eclipse; you can do that using the update site

http://download.eclipse.org/tools/buckminster/updates-4.2

You need to install the core, git and pde features.

Install Buckminster features

Prepare an empty target platform

It is better to start with an empty target platform in your IDE (then Buckminster will materialize the required features and bundles starting from that empty target); you then need to create an empty target platform manually that contains one single and empty directory and set this target platform active. A subsequent resolution/materialization will use that platform. Here are the detailed steps:

  1. Create a new general project named TP (or some name of your preference)
  2. Open "Window" => "Preferences" and expand "Plug-in Development"
  3. Select "Target Platform"
  4. Click "Add..."
  5. Click "Next>" (i.e. start with nothing)
  6. Enter TP in the Name: field
  7. Click "Add..."
  8. Select "Directory" and click "Next"
  9. Click on "Variables..."
  10. Scroll down and select "workspace_loc" and then type TP in the Argument: field.
  11. Click "OK" and then "Finish" and then "Finish" again.
  12. Set your new target definition active by checking it in the list of Target definitions: and click "OK"

Materialize the Workspace

From the File menu choose Open a Component Query... and use this URL

http://sourceforge.net/p/xsemantics/code/ci/master/tree/releng/it.xsemantics.buckminster/remote/web-build.cquery?format=raw

The component query opened

(Optional) Customize some Properties

Before starting the materialization, you might want to customize some properties; you can do that in the Properties tab of the opened component query:

The properties tab

You can add a property by pressing the New button.

You can customize the path in your system where the git repository will be cloned (property git.clone.dest), which defaults to ${user.home}/git/xsemantics): you can change it to something else, for instance, ${user.home}/repositories/xsemantics

Start Materialization

You can now press the button in the opened query Resolve and Materialize. This will start the materialization process which will consist of cloning the git repository, materialize the target platform and importing all the projects into the workspace.

Materializing the target platform may take sometime (depending on your network connection), so you might want to go and get a coffee in the meantime ;)

When the whole process terminates, you will have a working workspace with Xsemantics projects and the needed target platform (thus, you should have no error!).
